# ⚽ FC PIR 팀 관리 시스템

공평한 팀 편성과 회비 관리를 한번에! 순수 정적 웹사이트로 구현된 풋살 팀 관리 시스템입니다.

## 🌟 주요 기능

### 👥 팀 편성
- **자동 밸런싱 알고리즘**: 포지션(공격/수비/올라운더)과 실력 레벨을 고려한 공평한 팀 구성
- **실시간 통계**: 팀별 레벨 합계, 포지션 분포, 밸런스 품질 표시
- **팀 히스토리**: 과거 팀 편성 기록 저장 및 조회

### 💰 회비 관리
- **엑셀 스타일 인터페이스**: 회원별 × 월별 매트릭스 형식으로 한눈에 확인
- **납부 상태 관리**: 납부 완료/미납 상태를 색상으로 구분
- **자동 통계**: 납부율, 총 납부액, 월별 합계 자동 계산
- **유연한 금액 설정**: 회원별, 월별 다른 금액 설정 가능
  - 기본 회비: 35,000원
  - 1월 회비 (입회비 포함): 50,000원

### 📸 갤러리
- **사진 업로드**: 관리자가 팀 활동 사진 업로드
- **그리드 뷰**: 반응형 그리드 레이아웃으로 사진 표시
- **이미지 모달**: 사진 클릭 시 큰 이미지로 확대
- **제목/설명**: 각 사진에 제목과 설명 추가 가능

### 👤 멤버 관리
- **CRUD 기능**: 멤버 추가, 수정, 삭제
- **상세 정보**: 이름, 포지션(ATT/DEF/ALL), 실력 레벨(1-10), 활성 상태
- **필터링**: 활성/비활성 회원 필터

### 🔐 관리자 기능
- **로그인 시스템**: 관리자 전용 기능 접근 제어
- **대시보드**: 주요 통계 및 빠른 액션
- **완전한 관리 권한**: 모든 데이터 관리 가능

## 🚀 기술 스택

- **Frontend**: HTML5, CSS3, Vanilla JavaScript (ES6+)
- **Storage**: LocalStorage (브라우저 기반 클라이언트 저장소)
- **Hosting**: Netlify (정적 사이트 호스팅)
- **No Build**: 빌드 도구 없이 순수 정적 파일로 동작

## 🎮 사용 방법

### 일반 사용자

1. **홈페이지 접속**
   - 전체 통계 및 주요 기능 확인

2. **팀 편성**
   - 참가할 멤버 선택
   - "팀 자동 편성" 버튼 클릭
   - 공평하게 나눠진 A팀 / B팀 확인

3. **회비 현황 확인**
   - 월별 납부 현황을 엑셀 스타일로 확인
   - 연도 선택 가능
   - 회원별 합계, 월별 합계 확인

4. **갤러리 보기**
   - 업로드된 팀 활동 사진 확인
   - 사진 클릭으로 큰 이미지 보기

### 관리자

1. **로그인**
   ```
   ID: admin
   PW: admin1234
   ```

2. **멤버 관리**
   - 새 멤버 추가 (이름, 포지션, 레벨)
   - 기존 멤버 수정/삭제
   - 활성/비활성 상태 관리

3. **회비 관리**
   - 엑셀 스타일 테이블에서 셀 클릭
   - 빈 셀: 회비 등록 (1월 50,000원, 나머지 35,000원)
   - 미납 셀: 납부 처리
   - 납부 완료 셀: 미납 처리

4. **갤러리 관리**
   - 이미지 업로드 (JPG, PNG, GIF)
   - 제목/설명 입력
   - 업로드된 사진 삭제

## 🔧 로컬 개발 환경 설정

### 1. 프로젝트 클론
```bash
git clone https://github.com/your-username/fc-pir.git
cd fc-pir
```

### 2. 로컬 서버 실행
아래 방법 중 하나 선택:

**방법 1: Python 서버**
```bash
# Python 3
python -m http.server 8000

# 브라우저에서 http://localhost:8000 접속
```

**방법 2: Node.js serve**
```bash
npx serve
```

**방법 3: VS Code Live Server**
- VS Code에서 `index.html` 우클릭
- "Open with Live Server" 선택

### 3. 데이터 초기화
- 브라우저 개발자 도구 (F12)
- Application → Local Storage
- `football_*` 키 모두 삭제
- 페이지 새로고침 → 샘플 데이터 자동 생성

## 🌐 배포 (Netlify)

## 🎯 팀 밸런싱 알고리즘

### 동작 원리
1. **그룹화**: 멤버를 포지션별로 분류 (ATT/DEF/ALL)
2. **정렬**: 각 그룹을 레벨 순으로 정렬 (내림차순)
3. **지그재그 분배**:
   - 0, 2, 4, 6... 인덱스 → A팀
   - 1, 3, 5, 7... 인덱스 → B팀
4. **반복 최적화** (최대 100회):
   - 레벨 차이 계산
   - 같은 포지션끼리 스왑 시도
   - 개선되면 적용, 아니면 롤백
5. **결과 출력**: 최종 밸런스 통계

### 밸런스 품질
- **레벨 차이 ≤ 2**: 우수 (초록색)
- **레벨 차이 ≤ 5**: 양호 (파란색)
- **레벨 차이 ≤ 10**: 보통 (노란색)
- **레벨 차이 > 10**: 불량 (빨간색)

## ⚠️ 제한사항

### LocalStorage 용량
- **브라우저별 한도**: 5-10MB
- **권장사항**:
  - 이미지는 웹 최적화 버전 사용
  - 정기적으로 오래된 데이터 정리
  - 중요 데이터는 별도 백업

### 데이터 영속성
- **브라우저 캐시 삭제 시 데이터 손실**
- **다른 브라우저/기기에서 데이터 공유 불가**
- **실제 서비스에서는 백엔드 + DB 권장**

### 보안
- **클라이언트 사이드 인증**: 실제 보안 수준 낮음
- **개발/학습 목적으로 적합**
- **프로덕션에서는 서버 인증 필요**
